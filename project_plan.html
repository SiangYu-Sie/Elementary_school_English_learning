方案一：RPG 冒險戰鬥模式 (先期實作)

技術實作與無縫擴充架構

為了保證方案一能快速開發上線，並在後續能無痛融入方案二，我們需要設計一個高度模組化的 React + Supabase 資料庫架構。

React 組件模組化設計

  • 學習地圖組件 (GameMap)：渲染關卡節點，串接 Supabase 單元狀態。
  • 戰鬥測驗引擎 (BattleEngine)：承襲原 `QuizEngine` 的狀態機，增加怪物血條、傷害與音效邏輯。
  • 卡牌元件 (VocabularyCard)：具備 3D 翻牌動畫與解鎖/屬性顯示，預留擴充接口。

Supabase 資料庫結構擴充

  • profiles (核心)：擴充 `coins` (金幣), `level` (冒險等級), `exp` (經驗值) 欄位。
  • user_vocab_status：記錄每個單字的熟練度與是否卡牌化 (`is_card_unlocked`)。
  • user_pets (方案二擴充)：新表，儲存寵物類型、進化等級、總獲得經驗值。

實作串接流程

1. 擴充 profiles 欄位與單字狀態表 (Supabase 遷移)
2. 開發方案一：整合 WordCard 與 QuizEngine，推出「RPG 冒險挑戰關卡」
3. (後續擴充) 啟用 VocabularyCard 圖鑑，開啟「寵物養成進化」功能
